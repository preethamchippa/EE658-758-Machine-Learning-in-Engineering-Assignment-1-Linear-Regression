class NeuralNetwork():
    def __init__(self, learning_rate = 1e-3, epochs=10000):
        self.learning_rate = learning_rate
        self.epochs = epochs

    def forward(self, X, W1, b1, W2, b2):
        Z = 1 / (1 + np.exp(-X.dot(W1) - b1))
        A = Z.dot(W2) + b2
        expA = np.exp(A)
        Y = expA / expA.sum(axis=1, keepdims=True)
        return Y, Z


    # determine the classification rate
    # num correct / num total
    def classification_rate(self, Y, P):
        n_correct = 0
        n_total = 0
        for i in range(len(Y)):
            n_total += 1
            if Y[i] == P[i]:
                n_correct += 1
        return float(n_correct) / n_total


    def derivative_w2(self, Z, T, Y):
        N, K = T.shape
        M = Z.shape[1] # H is (N, M)

        ret4 = Z.T.dot(T - Y)

        return ret4


    def derivative_w1(self, X, Z, T, Y, W2):
        N, D = X.shape
        M, K = W2.shape

        dZ = (T - Y).dot(W2.T) * Z * (1 - Z)
        ret2 = X.T.dot(dZ)

        # assert(np.abs(ret1 - ret2).sum() < 0.00001)

        return ret2


    def derivative_b2(self, T, Y):
        return (T - Y).sum(axis=0)


    def derivative_b1(self, T, Y, W2, Z):
        return ((T - Y).dot(W2.T) * Z * (1 - Z)).sum(axis=0)


    def cost(self, T, Y):
        tot = T * np.log(Y)
        return tot.sum()


    def fit(self, X, Y):
        # create the data
        D = X.shape[1] # dimensionality of input
        M = 10 # hidden layer size
        K = len(set(Y)) # number of classes


        N = len(Y)
        # turn Y into an indicator matrix for training
        T = np.zeros((N, K))
        for i in range(N):
            T[i, Y[i]] = 1



        # randomly initialize weights
        W1 = np.random.randn(D, M)
        b1 = np.random.randn(M)
        W2 = np.random.randn(M, K)
        b2 = np.random.randn(K)

        costs = []
        for epoch in range(self.epochs):
            output, hidden = self.forward(X, W1, b1, W2, b2)
            if epoch % 1000 == 0:
                c = self.cost(T, output)
                P = np.argmax(output, axis=1)
                r = self.classification_rate(Y, P)
                print("cost:", c, "classification_rate:", r)
                costs.append(c)

            # this is gradient ASCENT, not DESCENT
            # be comfortable with both!
            # oldW2 = W2.copy()

            gW2 = self.derivative_w2(hidden, T, output)
            gb2 = self.derivative_b2(T, output)
            gW1 = self.derivative_w1(X, hidden, T, output, W2)
            gb1 = self.derivative_b1(T, output, W2, hidden)

            W2 += self.learning_rate * gW2
            b2 += self.learning_rate * gb2
            W1 += self.learning_rate * gW1
            b1 += self.learning_rate * gb1

        return W1, b1, W2, b2


    def predict(self, X, W1, b1, W2, b2):
        Y, _ = self.forward(X, W1, b1, W2, b2)
        predictions = np.argmax(Y, axis=1)
        return predictions

Hidden Layers and Network Performance: A Balancing Act¶

I explored how the number of neurons in hidden layers impacts a neural network's ability to learn. The results suggest a fascinating trend it's a balancing act!

Initially, as we ramp up the number of neurons, the network's accuracy climbs. This makes sense; the model gains more complexity, allowing it to capture subtler patterns within the data. More neurons translate to a more powerful tool for representing the intricate relationships between features and the desired output.

But there's a catch. Keep adding neurons, and accuracy starts to dip. This seems to be caused by overfitting. With too much power, the network becomes overly flexible. Instead of learning generalizable rules, it starts memorizing the training data essentially a case of cramming for a test it'll never see again.

The sweet spot, the optimal number of neurons, depends on a few factors. The data's complexity plays a role more intricate data likely needs more neurons to untangle its secrets. The size of the training set matters too. With a larger dataset, the model can handle more parameters (weights and biases) effectively, potentially pushing the optimal neuron count higher. Finally, the network architecture itself is important. Deeper networks, with more hidden layers stacked on top of each other, can often learn complex patterns with fewer neurons per layer compared to shallower networks.

Finding the ideal number of neurons is an art as much as a science. It involves experimentation trial and error with different configurations. Techniques like validation sets become crucial in identifying when overfitting rears its ugly head.

Overall, this exploration highlights the importance of finding the right balance. Too few neurons, and the model underfits, missing crucial patterns. Too many, and it overfits, becoming a champion on training data but a flop on unseen examples. The goal is to create a network that's powerful enough to learn but not so powerful it forgets how to learn in general.

5.Compare the performance of the neural network models with logistic regression, discussing the strengths and weaknesses of each approach.

Logistic Regression:

Strengths: Logistic regression stands out for its simplicity and ease of interpretation. Its straightforward nature makes it accessible for users with varying levels of expertise. Additionally, it performs efficiently, particularly on smaller datasets where classes are linearly separable. Moreover, the algorithm's interpretability is a notable advantage, as it provides clear coefficients for each feature, facilitating insight into their individual impacts on predictions.

Weaknesses: One of the primary limitations of logistic regression lies in its assumption of linear relationships between features and the log-odds of the target variable. This assumption can constrain its ability to capture complex, non-linear relationships within the data. Furthermore, effective deployment often requires extensive feature engineering to represent non-linear relationships adequately. Additionally, logistic regression may struggle with complex datasets, exhibiting high bias and potential underfitting when confronted with non-linear relationships between features and the target variable.

Neural Network Models:

Strengths: Neural networks offer remarkable flexibility in capturing intricate non-linear relationships, owing to their hierarchical architecture and non-linear activation functions. They excel in feature learning, autonomously extracting meaningful features from raw data, thereby reducing the need for extensive manual feature engineering. Furthermore, neural networks demonstrate scalability, effectively handling large datasets and high-dimensional feature spaces.

Weaknesses: The complexity inherent in neural networks poses challenges, particularly regarding interpretation and susceptibility to overfitting, especially when dealing with smaller datasets. Moreover, training neural networks can be computationally intensive, necessitating substantial computational resources, particularly for deep architectures or large datasets. Additionally, hyperparameter tuning is crucial but time-consuming, demanding expertise to optimize parameters such as learning rate, layer architecture, and neuron count effectively.

Comparison:

Performance: Neural networks typically outperform logistic regression, particularly on tasks characterized by complexity, large datasets, and non-linear relationships.

Interpretability: Logistic regression offers clear interpretability through its coefficients, making it preferable for scenarios prioritizing interpretability over complexity. In contrast, neural networks lack interpretability due to their intricate structure and numerous parameters.

Computational Resources: Logistic regression is computationally efficient and requires fewer resources compared to neural networks, making it suitable for resource-constrained environments or real-time applications.

Data Size: Logistic regression performs well on small to medium-sized datasets with linearly separable classes, whereas neural networks excel with larger datasets or when intricate patterns need to be discerned.

In essence, the choice between logistic regression and neural networks hinges on various factors, including dataset characteristics, task complexity, available computational resources, and interpretability requirements. Logistic regression is preferred for simpler tasks, linear relationships, and interpretability needs, while neural networks are favored for complex tasks, non-linear relationships, and scalability.
